**分析**：


**推测原因**: 

- 可能某些turn没有工具调用（纯文本响应），导致iterations计数<3
- 或者配置的阈值判断逻辑有边界问题

**建议**: 

1. 检查TC-29的服务端日志，确认实际的iters和tool_calls计数
2. 如果是边界case，考虑调整为：`reviewTriggerIterations: 3` 且 `reviewMinToolCalls: 3`（当前是AND逻辑，可能需要OR）
